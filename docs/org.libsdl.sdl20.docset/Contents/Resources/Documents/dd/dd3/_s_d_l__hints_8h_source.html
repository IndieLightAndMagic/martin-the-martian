22 /**
23  * \file SDL_hints.h
24  *
25  * Official documentation for SDL configuration variables
26  *
27  * This file contains functions to set and get configuration hints,
28  * as well as listing each of them alphabetically.
29  *
30  * The convention for naming hints is SDL_HINT_X, where "SDL_X" is
31  * the environment variable that can be used to override the default.
32  *
33  * In general these hints are just that - they may or may not be
34  * supported or applicable on any given platform, but they provide
35  * a way for an application or user to give the library a hint as
36  * to how they would like the library to work.
37  */

51  * \brief A variable controlling how 3D acceleration is used to accelerate the SDL screen surface.
52  *
53  * SDL can try to accelerate the SDL screen surface by using streaming
54  * textures with a 3D rendering engine. This variable controls whether and
55  * how this is done.
56  *
57  * This variable can be set to the following values:
58  * "0" - Disable 3D acceleration
59  * "1" - Enable 3D acceleration, using the default renderer.
60  * "X" - Enable 3D acceleration, using X where X is one of the valid rendering drivers. (e.g. "direct3d", "opengl", etc.)

62  * By default SDL tries to make a best guess for each platform whether
63  * to use acceleration or not.
64  */

617 /** \brief Allows back-button-press events on Windows Phone to be marked as handled
618  *
619  * Windows Phone devices typically feature a Back button. When pressed,
620  * the OS will emit back-button-press events, which apps are expected to
621  * handle in an appropriate manner. If apps do not explicitly mark these
622  * events as 'Handled', then the OS will invoke its default behavior for
623  * unhandled back-button-press events, which on Windows Phone 8 and 8.1 is to
624  * terminate the app (and attempt to switch to the previous app, or to the
625  * device's home screen).
626  *
627  * Setting the SDL_HINT_WINRT_HANDLE_BACK_BUTTON hint to "1" will cause SDL
628  * to mark back-button-press events as Handled, if and when one is sent to
629  * the app.
630  *
631  * Internally, Windows Phone sends back button events as parameters to
632  * special back-button-press callback functions. Apps that need to respond
633  * to back-button-press events are expected to register one or more
634  * callback functions for such, shortly after being launched (during the
635  * app's initialization phase). After the back button is pressed, the OS
636  * will invoke these callbacks. If the app's callback(s) do not explicitly
637  * mark the event as handled by the time they return, or if the app never
638  * registers one of these callback, the OS will consider the event
639  * un-handled, and it will apply its default back button behavior (terminate
640  * the app).
641  *
642  * SDL registers its own back-button-press callback with the Windows Phone
643  * OS. This callback will emit a pair of SDL key-press events (SDL_KEYDOWN
644  * and SDL_KEYUP), each with a scancode of SDL_SCANCODE_AC_BACK, after which

646  * If the hint's value is set to "1", the back button event's Handled
647  * property will get set to 'true'. If the hint's value is set to something
648  * else, or if it is unset, SDL will leave the event's Handled property
649  * alone. (By default, the OS sets this property to 'false', to note.)
650  *
651  * SDL apps can either set SDL_HINT_WINRT_HANDLE_BACK_BUTTON well before a
652  * back button is pressed, or can set it in direct-response to a back button
653  * being pressed.
654  *
655  * In order to get notified when a back button is pressed, SDL apps should
656  * register a callback function with SDL_AddEventWatch(), and have it listen
657  * for SDL_KEYDOWN events that have a scancode of SDL_SCANCODE_AC_BACK.
658  * (Alternatively, SDL_KEYUP events can be listened-for. Listening for
659  * either event type is suitable.) Any value of SDL_HINT_WINRT_HANDLE_BACK_BUTTON
660  * set by such a callback, will be applied to the OS' current
661  * back-button-press event.
662  *
663  * More details on back button behavior in Windows Phone apps can be found
664  * at the following page, on Microsoft's developer site:
665  * http://msdn.microsoft.com/en-us/library/windowsphone/develop/jj247550(v=vs.105).aspx
